Задание

Реализовать простой REST сервис для игры в крестики-нолики.
Условие: сервис должен представлять собой Node.js (Mongoose, Express) приложение и использовать MongoDB для хранения информации об играх.

Сервис должен поддерживать следующие запросы:
GET /games/ - возвращает список игр
GET /games/:id - возвращает информацию об игре (по id)
POST /games/ - создает новую игру и возвращает ее информацию
PUT /games/:id - выполняет ход игрока в игре
DELETE /games/:id - удаляет игру по id

Ходы(запросы) в игре должны следовать правилам игры крестики-нолики:
игроки могут посылать ходы только последовательно
когда крестики или нолики выстраиваются в ряд из трех, игра считается завершенной (имеется победитель)
если доступных ходов более не имеется и победитель не выявлен, игра считается завершенной (ничья)

Плюсом будет использование публичного репозитория (bitbucket.com, github.com) для решения.
Валидация и юниттесты необязательны.
